# Тестовое задание — парсер Wildberries

Привет! Это моё тестовое задание. Я написал парсер, который собирает товары с Wildberries по поисковому запросу и сохраняет их в Excel-файлы.

## Что делает парсер

- Ищет товары по запросу **"пальто из натуральной шерсти"**
- Собирает по каждому товару: ссылку, артикул, название, цену, описание, фото, продавца, размеры, остатки, рейтинг, отзывы и все характеристики
- Сохраняет полный каталог в `catalog_full.xlsx`
- Отдельно делает выборку товаров с рейтингом ≥ 4.5, ценой до 10 000 ₽ и страной производства Россия — в `catalog_filtered.xlsx`

## Как запустить

Установить зависимости:

```bash
pip install -r requirements.txt
```

Запустить парсер:

```bash
python src/parser.py
```

Можно передать свой запрос или количество страниц:

```bash
python src/parser.py --query "ваш запрос" --pages 10
```

Результаты появятся в папке `output/`.

## Готовые результаты

Я уже положил готовые файлы в папку `output/`, можно сразу посмотреть:

- `output/catalog_full.xlsx` — 15 товаров, полный каталог
- `output/catalog_filtered.xlsx` — 8 товаров, прошедших фильтр

## Про блокировки Wildberries

Wildberries блокирует автоматические запросы и возвращает 429. Я пробовал разные способы обойти это (в том числе `curl_cffi` с подменой TLS-отпечатка под браузер), но поисковый API всё равно блокирует.

Поэтому я добавил запасной вариант: если API недоступен, парсер автоматически берёт данные из `data/sample_data.json` и всё равно генерирует оба Excel-файла. Логика фильтрации, структура характеристик и формат вывода — всё работает так же, как на реальных данных.

## Стек

- Python 3.9+
- `curl_cffi` — запросы с имперсонацией браузера
- `openpyxl` — генерация xlsx
